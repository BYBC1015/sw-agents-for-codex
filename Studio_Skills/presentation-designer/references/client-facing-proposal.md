# Client-Facing Proposal Deck Reference

Use this reference when the deck is meant for a client, boss, investor, government department, public-sector stakeholder, or formal approval meeting.

## Core Principle

A proposal deck is not a container for all information. It arranges the audience's judgment path:

1. What decision or approval is needed?
2. What is the real problem or opportunity?
3. Why is this direction credible?
4. What is the core proposition?
5. How will it be executed?
6. What will be delivered next?

## Client Goal First

Before structure or visual design, identify the work result:

- approve budget
- approve creative direction
- align departments
- build public-sector confidence
- win a pitch
- explain a production plan
- support investment or tourism promotion
- make a leader comfortable presenting upward

If unclear, ask one concise question or proceed with a stated assumption.

## Research Rule

Research should serve a decision, not prove effort.

- Keep research to 1-2 pages by default.
- Each research page must lead to a project judgment.
- Do not turn a creative or production proposal into a market report unless the user asks.
- After data, immediately state the task it creates for the project.

## Structure Pattern

For formal proposal decks, use this pattern as a starting point:

```text
Cover
Agenda / Table of contents
Project judgment
Communication or business task
Positioning
Creative proposition
Content / solution structure
Execution method
Visual / production system
Delivery standard
Next step
```

Keep the final page order flexible. The goal is a clear judgment path, not template obedience.

## Client Language

Avoid internal or rough production words in client-facing pages:

- internal
- client requirement breakdown
- unified reporting language
- Party A /甲方
- AI
- what we want to shoot
- placeholder
- temporary draft
- customer said

Prefer formal, client-ready labels:

- project judgment
- communication task
- strategic opportunity
- positioning
- creative proposition
- content structure
- image method
- delivery standard
- next-step recommendation

## Title Rules

Weak titles name a topic. Strong titles make a judgment.

Weak:

- Project understanding
- Market analysis
- Shooting plan
- Visual reference

Stronger:

- A growing market window gives the film a clearer brand task
- This is not a scenic ad, but a city image film
- Four minutes is not a longer runtime, but a complete emotional progression
- One main film should become a reusable visual asset system

## Markdown-First Rule

For decks longer than 8 slides, first write a Markdown outline:

- one slide title
- one short explanation
- 2-4 support points
- visual role
- source/asset need

Only move into PPT layout after the storyline is accepted or reasonably assumed.
