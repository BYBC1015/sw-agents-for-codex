# 新项目启动语

以后每次新建对话，先调用：

```text
/SW_V1.0
```

或：

```text
@SW_V1.0
```

调用成功后，Codex 应只回复：

```text
SW loaded successfully. Current version: V1.3.5
```

然后你再输入项目诉求即可。Codex 会自动判断这是海报、PPT/deck、视频，还是海报+视频，并按本项目的设计工作室架构执行。

## 对话中常用短句

- `继续`：按当前阶段推进。
- `你定`：让 Codex 按行业默认值继续。
- `少问`：每轮最多问 1 个问题。
- `收短`：压缩输出，节省 token。
- `展开`：把当前方案细化。
- `出海报方案`：进入海报方案。
- `出PPT方案` / `出PPT大纲`：进入 presentation/deck 结构与页面角色。
- `出分镜`：进入视频分镜。
- `出剪辑表`：进入剪辑方案。
- `出图片提示词`：按图片提示词规则输出。
- `出视频提示词`：按视频镜头提示词规则输出。
- `终审`：只检查风险和必改项。

示例：

```text
/SW_V1.0
```

```text
做一个新品咖啡小红书海报，高级但年轻，不要太花。
```

也可以一条消息写完：

```text
/SW_V1.0 做一个艺术市集 30 秒短视频，发抖音和视频号，要有城市感。
```

此时 Codex 应先回复成功语，再继续执行项目。

```text
做一个艺术市集 30 秒短视频，发抖音和视频号，要有城市感。
```

## 工作模式

- `省 token 模式`：只给结论、关键假设、下一步。
- `标准模式`：默认模式，适合日常项目。
- `精修模式`：方向已定后输出更完整执行版。
- `公司交付模式`：增加汇报结构、风险和交接话术。

## 最小输入

你不用一次性写很完整。至少给下面任意 2-3 项即可：

- 要做什么：海报 / PPT-deck / 视频 / 海报+视频
- 发在哪：小红书 / 抖音 / 视频号 / 朋友圈 / 线下 / 其他
- 主题或原始需求
- 必须出现的信息
- 不想要的风格
- 截止时间

## 推荐输入例子

```text
/SW_V1.0
```

```text
我们要做一个新品咖啡海报和 15 秒短视频，发小红书和朋友圈。老板想要高级但年轻，不要太花，要突出新品上市和限时优惠。
```

```text
@SW_V1.0
```

```text
做一个艺术市集 30 秒短视频，发抖音和视频号，风格轻松、有城市感，希望看完让大家想来现场。
```
