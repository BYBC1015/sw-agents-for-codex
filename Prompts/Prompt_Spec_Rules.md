# 图片与视频提示词生成规范

本文件用于约束 Codex 生成 AI 图片、AI 视频、分镜图、海报主视觉、动态镜头提示词时的写法。目标是把“近一点、远一点、高级一点、年轻一点、炸一点”这类模糊表达，翻译成可执行的画面参数。

## 核心规则

- 不输出只有形容词的提示词。`高级、电影感、年轻、炸、氛围感、近一点、远一点` 都必须转成具体视觉参数。
- 每条提示词必须说明：主体、场景、构图、景别、镜头角度、光线、色彩、风格、用途、画幅、限制。
- 视频提示词必须额外说明：时长、动作、镜头运动、节奏、起始画面、结束画面、主体连续性、禁止变形。
- 如果用户给的信息不足，先按常用默认值生成一版，并标注默认假设，不要反复追问。
- 若用于商业交付，避免要求生成真实名人、品牌 logo、受版权保护角色、仿某在世艺术家的风格。
- 先输出通用提示词，再按 `Tool_Registry.md` 转成工具专用提示词；没有工具卡时，交给 `tool-integrator` 注册工具。
- Banana Pro / `banana-pro` 是提示词后的图像生成成员，作用类似 `imagegen`；只在用户明确要求香蕉、banana、Banana Pro、Nano Banana 或 OpenRouter 图像生成时调用，普通图片提示词不要自动切换到 Banana Pro。
- 用户同时要求“设计分镜/海报/PPT 页面/角色/产品/场景”和“使用 banana 出图”时，不能越过上游设计环节直接生成。先由对应 Studio 角色锁定分镜、版式、页面角色或视觉 brief，再由 `prompt-director` 转成可执行提示词，最后交给 `banana-pro` 生成选中的一张或一帧。

## 工具适配层

提示词必须分成两层，方便未来接入新工具：

```text
通用版 Prompt：
[不绑定工具，完整描述创意意图和视觉参数]

工具版 Prompt：
[按 Tool_Registry.md 中的工具能力、参数、负面词、参考图规则转换]

参数锁定：
[主体、产品、光线、色彩、画幅、一致性]

生成后质检：
[文字/logo、产品形状、版权相似、平台规格、可排版性]
```

如果用户指定工具：

- 先查 `Tool_Registry.md` 是否有工具卡。
- 再按 `AI_Tool_Prompt_Profiles.md` 选择对应工具剖面。
- 有工具卡：按该工具的提示词格式输出。
- 无工具卡：先用 `tool-integrator` 建工具卡，再输出工具版。
- 工具参数可能更新时，查官方说明或让用户提供工具说明。

## 先判断用途

生成提示词前，先判断用户要的是哪一种：

| 用途 | 重点 | 不要做 |
| --- | --- | --- |
| 海报主视觉 | 留白、信息层级、主体冲击力、可排版空间 | 不要让 AI 直接生成大量文字 |
| 海报背景 | 氛围、色彩、空间、低干扰 | 不要抢主标题 |
| 产品图 | 材质、光线、比例、细节真实 | 不要夸张变形产品 |
| 人物图 | 年龄、姿态、表情、服装、授权风险 | 不要生成真实名人替身 |
| 分镜图 | 镜头可读、动作清楚、连续性 | 不要过度精修 |
| 视频镜头 | 起止状态、动作、运镜、节奏 | 不要只写“动起来” |
| 视频整体 | 分镜表和节奏 | 不要把多个复杂镜头塞进一句 prompt |

## 模糊词翻译表

| 用户说法 | 不能直接写 | 应翻译成 |
| --- | --- | --- |
| 近一点 | closer | 景别从中景改为近景；主体占画面 60%-75%；裁切到胸口以上；背景虚化加强 |
| 远一点 | farther | 景别从近景改为全景/远景；主体占画面 10%-25%；保留环境和空间关系 |
| 高级一点 | premium | 元素减少；留白增加；低饱和配色；柔和定向光；材质细节；信息层级克制 |
| 年轻一点 | young | 更高对比；动态构图；明亮点缀色；生活化道具；更轻快的姿态或节奏 |
| 炸一点 | explosive | 大尺度主体；强对比；前景冲击；快速运动；强光边缘；卡点节奏 |
| 干净一点 | clean | 背景元素减少；色彩不超过 3 个主色；清晰边界；更大负空间 |
| 电影感 | cinematic | 具体焦段、景深、横向层次、自然光或动机光、阴影层次、画面纵深 |
| 真实一点 | realistic | 真实材质瑕疵；自然皮肤/产品纹理；合理透视；真实镜头景深；避免过度磨皮 |
| 梦幻一点 | dreamy | 柔焦、高光晕染、浅景深、低对比、轻雾、柔和渐变色 |
| 科技感 | tech | 冷暖对比、精密材质、网格/界面光、边缘光、玻璃/金属、秩序感构图 |
| 不要太花 | less busy | 删除装饰元素；降低图案密度；减少颜色和字体；强化一个视觉中心 |

## 景别与距离规则

用景别、主体占比和裁切位置代替“近/远”。

| 景别 | 主体占画面 | 常见裁切 | 适合用途 |
| --- | --- | --- | --- |
| 特写 ECU | 80%-100% | 局部细节、眼睛、产品纹理 | 情绪、质感、细节 |
| 近景 CU | 60%-80% | 头肩、人脸、产品主体 | 人物情绪、产品卖点 |
| 中近景 MCU | 45%-60% | 胸口以上或产品+少量环境 | 口播封面、主视觉 |
| 中景 MS | 30%-45% | 腰部以上或产品+桌面 | 人物动作、场景关系 |
| 全景 FS | 15%-30% | 全身或完整产品陈列 | 造型、活动、空间 |
| 远景 WS | 5%-15% | 人/产品在环境中 | 氛围、场地、品牌调性 |
| 大远景 EWS | 5% 以下 | 环境为主 | 建立场景、开场镜头 |

可附加焦段，但不要滥用：

| 焦段 | 视觉效果 | 适合 |
| --- | --- | --- |
| 24mm | 空间更夸张，环境更多 | 建筑、活动、人群、运动感 |
| 35mm | 自然、有现场感 | 生活方式、街头、场景人像 |
| 50mm | 接近自然视角 | 产品、人物、通用主视觉 |
| 85mm | 背景压缩，主体突出 | 人像、高级质感、浅景深 |
| 100mm macro | 细节放大 | 食品、材质、产品纹理 |

## 图片提示词结构

每次输出图片提示词，优先给“一条主提示词 + 一条负面提示词 + 可控修改项”，不要默认给 10 个版本。

### 图片主提示词必须包含

1. 用途：海报主视觉、背景、产品图、分镜图等。
2. 主体：谁/什么，数量，状态，动作。
3. 场景：地点、时间、空间层次、道具。
4. 构图：主体位置、主体占比、留白方向、信息区。
5. 景别：特写、近景、中景、全景、远景。
6. 镜头：角度、焦段或视觉效果、景深。
7. 光线：光源方向、软硬、对比、高光和阴影。
8. 色彩：主色、辅助色、饱和度、冷暖。
9. 风格：摄影、3D、插画、拼贴等，并描述具体材质。
10. 画幅：1:1、3:4、4:5、9:16、16:9 等。
11. 限制：不要文字、不要 logo、不要多余肢体、不要变形。

### 图片提示词模板

```text
用途：[海报主视觉/背景/产品图/分镜图]
主提示词：
[主体]，[动作/状态]，位于[场景]；[景别]，[主体占画面 xx%]，[构图方式]，[留白位置用于排版]；[镜头角度]，[焦段/景深效果]；[光线方向和质感]；[主色与辅助色]；[材质和风格]；[画幅]；画面干净，商业广告级质感。

负面提示词：
不要文字，不要错误 logo，不要水印，不要畸形手指，不要多余肢体，不要低清晰度，不要过度磨皮，不要杂乱背景，不要侵权角色。

可控修改项：
- 想更近：把景别改为[近景/特写]，主体占比提升到[xx%]。
- 想更远：把景别改为[全景/远景]，主体占比降低到[xx%]，增加环境。
- 想更高级：减少道具，增加留白，降低饱和度，强化材质和柔和定向光。
```

## 视频提示词结构

视频提示词不要只写一个漂亮场景。必须写清“时间内发生什么变化”。

### 视频主提示词必须包含

1. 镜头编号和时长。
2. 起始画面。
3. 主体动作。
4. 镜头运动。
5. 结束画面。
6. 景别和构图变化。
7. 光线和色彩连续性。
8. 运动速度和节奏。
9. 画幅和平台。
10. 禁止项：变形、跳切、主体变化、错误文字、多余元素。

### 视频镜头提示词模板

```text
镜头 [编号]，[时长]，[画幅/平台]。
起始画面：[主体] 位于 [场景]，[景别]，[主体占画面 xx%]，[构图]。
动作：[主体在 x 秒内做什么]。
镜头运动：[推近/拉远/横移/环绕/俯仰/固定]，从 [起点] 到 [终点]，速度 [慢/中/快]，运动平稳。
结束画面：[结束时主体位置、景别、表情/状态]。
视觉风格：[光线、色彩、材质、画面质感]。
连续性要求：[主体外观、服装、产品形态、背景保持一致]。
禁止：[不要变形、不要闪烁、不要突然换场景、不要错误文字、不要多余人物、不要水印]。
```

### 多镜头视频规则

- 15 秒以内：建议 3-6 个镜头。
- 30 秒以内：建议 6-10 个镜头。
- 每个镜头单独写提示词，避免一个 prompt 同时塞入太多动作。
- 第一镜头必须有钩子：冲突、动作、视觉奇观、问题或强信息。
- 每个镜头要能接上前后镜头：主体、色彩、空间或动作至少保持一个连续线索。

## 运镜翻译表

| 用户说法 | 应写成 |
| --- | --- |
| 镜头靠近 | 缓慢 push-in，3 秒内从中景变为近景，主体占比从 35% 到 65% |
| 镜头拉开 | pull-out，4 秒内从近景变为全景，露出完整环境 |
| 有高级感的移动 | 稳定慢速 dolly-in 或轻微 orbit，运动幅度小，避免手持晃动 |
| 更有冲击 | 快速 push-in 或 whip pan，配合前景遮挡和强节奏点 |
| 更自然 | 固定镜头或轻微手持，主体自己进入画面，减少炫技运镜 |
| 展示产品 | 45 度三分之四角度，慢速环绕 15-30 度，保持产品形状稳定 |
| 展示空间 | 广角远景，横向 truck movement，露出前景、中景、背景层次 |

## 修改提示词时的规则

用户给“近一点、远一点、再高级一点”等反馈时，不要重写全部。只改相关参数：

```text
我会只调整这 3 个参数：
1. 景别：中景 -> 近景
2. 主体占比：35% -> 65%
3. 背景：保留但虚化加强

其他风格、光线、色彩保持不变。
```

这样可以减少 token，也能减少生成结果漂移。

## 参考图使用规则

当用户提供参考图、截图、竞品图或风格图时，必须先判断参考图用途：

| 参考用途 | 应提取 | 应避免 |
| --- | --- | --- |
| 风格参考 | 色彩、光线、材质、构图节奏 | 直接复刻画面和独特元素 |
| 构图参考 | 主体位置、留白、信息区、景别 | 复制品牌、人物、产品 |
| 产品参考 | 形状、材质、比例、关键卖点 | 改变产品结构 |
| 人物参考 | 姿态、服装气质、年龄段 | 生成真实人物替身 |
| 分镜参考 | 镜头顺序、运镜、节奏 | 照搬可识别镜头 |

参考图提示词要写成：

```text
参考图只用于提取 [色彩/构图/光线/材质/节奏]，不要复制其中的品牌、人物、logo、文字、独特图案或受版权保护元素。
```

## 文字与 logo 规则

大多数图像/视频生成模型不擅长稳定生成准确文字。商业设计默认按以下规则：

- 海报标题、价格、日期、二维码、logo 默认后期在 Photoshop、Illustrator、Canva 或剪辑软件中添加。
- 图片 prompt 中只要求“保留干净文字区域”，不要要求模型生成准确中文标题。
- 如果必须生成文字，只生成极短、可容错的英文字母或占位符，并在终审中检查。
- logo 默认使用客户提供文件后期叠加，不让 AI 临摹或生成相似 logo。

文字区域描述示例：

```text
画面上方保留 35% 干净留白，用于后期添加中文标题；背景低干扰，无文字、无 logo、无水印。
```

## 一致性规则

用于系列海报、多镜头视频或多次改稿时，要锁定一致性参数，减少每次生成都变样。

### 人物一致性

记录：

- 年龄段
- 发型
- 服装
- 体型
- 表情气质
- 关键配饰
- 镜头风格

不要在后续 prompt 中随意改变这些参数。

### 产品一致性

记录：

- 产品形状
- 材质
- 颜色
- 包装比例
- 标签位置
- 关键卖点
- 不能变形的部分

产品提示词必须加入：

```text
保持产品形状、比例、材质和包装结构稳定，不改变标签位置，不生成错误文字。
```

### 系列视觉一致性

记录：

- 主色和辅助色
- 光线方向
- 背景材质
- 构图规律
- 字体和信息区位置
- 画面颗粒、锐度、对比度

## 批量生成规则

批量生成容易浪费 token 和导致方向漂移，默认只在主方向确认后进行。

- 一次最多生成 3 个变体。
- 每个变体只改变一个主变量：构图、色彩、景别、光线或道具。
- 不要同时改变主体、风格、色彩、构图和镜头。
- 输出变体时必须说明“变的是什么，不变的是什么”。

批量变体格式：

```text
共同锁定：
- 主体：
- 风格：
- 光线：
- 色彩：
- 画幅：

变体 A：只改构图
变体 B：只改光线
变体 C：只改景别
```

## 平台和画幅规则

如果用户没说平台，先按常用商业默认：

| 场景 | 默认画幅 | 说明 |
| --- | --- | --- |
| 小红书海报 | 3:4 或 4:5 | 适合信息卡和生活方式图 |
| 朋友圈海报 | 3:4 或 1:1 | 注意移动端可读性 |
| 抖音/视频号短视频 | 9:16 | 人物和字幕要避开底部按钮区 |
| B站/横版视频 | 16:9 | 更适合叙事和信息展示 |
| 线下海报 | 按印刷尺寸 | 需要出血、分辨率和 CMYK 检查 |

提示词必须说明画幅，但不要把平台 UI 元素画进画面。

## 提示词质检清单

生成前自检：

- [ ] 是否说明了用途？
- [ ] 是否有明确主体和动作？
- [ ] 是否有景别或主体占比？
- [ ] 是否有构图和留白？
- [ ] 是否有镜头角度或焦段效果？
- [ ] 是否有光线方向和色彩？
- [ ] 是否有画幅？
- [ ] 是否有负面提示词？
- [ ] 视频是否有起始画面、动作、结束画面？
- [ ] 修改需求是否只改必要参数？

## 省消耗策略

- 第一次只输出 1 条主提示词，不批量输出多版本。
- 如果用户还没选方向，只输出“提示词骨架”，不写完整高成本提示词。
- 修改时只输出差异参数，不重新生成整段。
- 分镜视频先输出表格，再按用户选中的镜头生成详细视频 prompt。
- 海报主视觉先生成背景/主体提示词，文字排版建议在设计软件完成。

## 输出语言规则

- 用户没有指定时，图片提示词默认输出中文主提示词；如果要给外部生成工具使用，可同时给英文版。
- 视频提示词默认中文即可；如果用户要复制到英文模型，再补英文版。
- 不要中英混杂在同一条主提示词里，避免语义重复和权重混乱。
