# 自动派单测试样例

## 测试目标

验证 Codex 能否根据项目类型、阶段和用户意图自动分配岗位 skill，而不需要用户手动点名。

## 测试 1：海报新项目

用户输入：

```text
按 AGENTS.md 跑一个新设计项目：
做一张新品咖啡小红书海报，高级但年轻，不要太花。
```

期望派单：

```text
项目类型：海报
阶段：Brief
主责：brief-strategist
下一棒：creative-director
```

## 测试 2：视频分镜

用户输入：

```text
继续，出分镜。
```

如果当前项目是视频或海报+视频，期望派单：

```text
阶段：分镜
主责：storyboard-director
下一棒：edit-director
```

## 测试 3：图片提示词

用户输入：

```text
给这个主视觉出图片提示词，咖啡杯近一点，高级一点。
```

期望派单：

```text
阶段：提示词
主责：prompt-director
必须读取：Prompts/Prompt_Spec_Rules.md
```

合格表现：

- 把“近一点”翻译成景别、主体占比、裁切和背景虚化。
- 把“高级一点”翻译成留白、低饱和、材质和柔和定向光。

## 测试 4：改稿

用户输入：

```text
老板说不够炸，想更适合年轻人传播。
```

期望派单：

```text
阶段：变更
主责：change-manager
下一棒：poster-art-director 或 storyboard-director，取决于项目类型
```

合格表现：

- 先判断 S/M/L/XL。
- 不直接重写全部。
- 说明哪些参数变，哪些保持。

## 测试 5：终审

用户输入：

```text
终审，看看能不能交付。
```

期望派单：

```text
阶段：终审
主责：review-producer
```

合格表现：

- 只检查事实、品牌、版权、平台、导出风险。
- 不新增创意方向。
