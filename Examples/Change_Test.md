# 改稿测试样例

## 测试目标

验证变更单机制能否判断影响范围、控制返工、保留旧版本。

## 原方案

```text
项目：新品咖啡小红书海报
方向：城市通勤里的精品咖啡
主视觉：清晨窗边咖啡杯、柔和自然光、少量城市倒影
文案：把清晨，喝成一种仪式感
```

## 新变更

```text
老板说：感觉太安静了，能不能更炸一点，更适合年轻人传播？
```

## 建议测试步骤

1. 把新变更写入 `Change_Log.md`。
2. 使用提示词 `14. 变更评估`。
3. 判断这是 M 还是 L：
   - 如果只加强色彩、文案和构图，是 M。
   - 如果从精品生活方式改成潮流派对咖啡，是 L。
4. 若为 M，只更新 `03_Poster/Poster_Workflow.md`。
5. 若为 L，回到 `02_Direction/Creative_Direction.md` 重新评估方向。
6. 在 `Token_Budget.md` 记录是否值得进入高成本重做。

## 验收标准

- 没有直接覆盖旧版。
- 能说清楚“更炸”对应到哪些设计动作。
- 能判断是否要回到创意方向阶段。
- 有给老板的解释话术。
