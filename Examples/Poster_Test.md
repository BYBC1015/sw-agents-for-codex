# 海报项目测试样例

## 测试目标

验证模板能否从一个模糊需求走到可执行海报方案，并控制方向数量和 token 消耗。

## 原始需求

```text
我们下周要做一个新品咖啡的海报，想要高级一点，但也要年轻人喜欢。主要发小红书和朋友圈，老板说要有质感，不要太花。
```

## 建议测试步骤

1. 把原始需求填入 `00_Brief/Brief.md`。
2. 使用提示词 `1. Brief 提取`。
3. 使用提示词 `3. 风格研究提炼`，假设参考方向为精品咖啡、城市通勤、浅奢生活方式。
4. 使用提示词 `4. 生成 3 个创意方向`。
5. 只选择 1 个方向进入 `03_Poster/Poster_Workflow.md`。
6. 使用提示词 `5. 海报文案生成` 和 `6. 海报版式建议`。
7. 在 `Token_Budget.md` 记录一次中成本任务。
8. 用 `06_FinalReview/Final_Review_Checklist.md` 检查是否能交付。

## 验收标准

- 创意方向不超过 3 个。
- 海报方案能明确主标题、视觉中心、色彩、字体和素材。
- 没有在 Brief 不清楚时直接生成大量最终稿。
- 有记录 token 消耗理由。
