# 工具接入测试样例

## 测试目标

验证新增工具时，Codex 能否先注册工具卡，再适配提示词和派单，而不是直接乱用工具。

## 测试 1：新增图片工具

用户输入：

```text
我新接了一个图片生成工具，可以输入中文 prompt，支持参考图和负面词，能出 3:4 和 9:16。
```

期望行为：

- 派单给 `tool-integrator`。
- 在 `Tool_Registry.md` 增加工具卡。
- 归属岗位为 `prompt-director`。
- 说明需要测试：参考图、负面词、画幅、文字稳定性、产品变形。

## 测试 2：新增视频工具

用户输入：

```text
我接了一个视频生成工具，最长 10 秒，支持首帧图和运镜描述。
```

期望行为：

- 派单给 `tool-integrator`。
- 工具类型为视频生成。
- 提示词必须逐镜头写。
- 要求记录：时长、画幅、首帧、运镜、主体一致性、禁止变形。

## 测试 3：工具提示词转换

用户输入：

```text
把这个通用海报主视觉 prompt 转成新图片工具可用版本。
```

期望行为：

- 派单给 `prompt-director`，必要时协作 `tool-integrator`。
- 输出通用版、工具版、负面词、参数锁定、质检项。
